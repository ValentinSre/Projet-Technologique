Exécute tes requêtes SQL

Visualise le contenu de la base de données

Clique sur une table pour en afficher le contenu.

En cliquant sur une table, tu afficheras son contenu ainsi qu'une explication de ses colonnes afin de faciliter ta manipulation.

Ce module ne renseigne que les informations présentes dans la base de données avant modification. L'affichage au sein de ce module ne prend pas en compte les éventuels ajouts/retraits que vous effectuerez sur la base de données.

  • Table adaptation

    Cette table se compose de 4 colonnes et 6 lignes.

  • Table artiste

La table livre se compose des colonnes suivantes :

titre qui désigne le titre du livre
annee qui désigne l'année de sortie du livre
genre qui désigne le genre du livre parmi Manga, Comics et Bande-dessinée
editeur qui désigne l'éditeur du livre
nomAuteur qui désigne le nom de famille de l'auteur du livre

Exemples

Inspire-toi de ces exemples

Ces exemples reviennent sur les requêtes basiques en s'inspirant des tables disponibles dans la base de données.

Projection

Affichage de toutes les colonnes de la table livre :

SELECT * FROM livre

Affichage de la colonne Titre de la table livre :

SELECT Titre FROM livre

Affichage des colonnes Titre, Genre et Annee de la table livre :

SELECT Titre, Genre, Annee FROM livre

Jointure

Jointure de deux tables (forme générique) :

SELECT * FROM table_A JOIN table_B ON tableA.clé = table_B.clé

Dans notre exemple, l'oeuvre de la table adaptation correspond au titre de la table livre.

SELECT * FROM livre JOIN adaptation ON livre.titre = adaptation.oeuvre

Sélection

Sélection des colonnes Titre de la table livre où le Genre est Comics

SELECT Titre FROM livre WHERE Genre = "Comics"

Sélection des colonnes Titre et Genre de la table livre où l'éditeur est Glénat

SELECT Titre, Genre FROM livre WHERE Editeur = "Glénat"

Création/suppression

Création d'une table acteur avec une colonne nom, prénom, dateNaissance, nationalite

CREATE TABLE acteur(nom text, prenom text, dateNaissance date, nationalite text)

Suppression de la table acteur

DROP TABLE acteur

Insertion

Insertion d'une ligne (entière) dans la table acteur

INSERT INTO acteur VALUES ("Holland", "Tom", 1996-06-01, "Royaume-Uni")

Insertion d'une ligne (partielle) dans la table acteur

INSERT INTO acteur ("prenom", "nationalite") VALUES ("Zendaya", "États-Unis")

Retrait

Suppression d'un (ou des) élément(s) d'une table (forme générique)

DELETE FROM table WHERE condition

Suppresion de la ligne "Tintin" de la table livre

DELETE FROM livre WHERE titre='Tintin'

AIDE

Construis ta requête SQL

Utilise ce guide pour structurer ta requête de la meilleure manière possible !

  • Structure globale d'une requête

    SELECT colonne(s)
    FROM Table1
    JOIN Table2 ON condition(s)
    WHERE condition(s)
    GROUP BY colonne(s)
    HAVING condition(s)
    ORDER BY colonne(s) ASC/DESC

  • Création de table

    CREATE TABLE nomTable (nomColonne1 type, nomColonne2 type, ...)

  • Remplissage de table

    INSERT INTO Table VALUES (valeur1, valeur2, ...)

  • Fonctions d'agrégation utiles

    SUM() : calcule la somme d'une colonne ou groupe de lignes
    AVG() : calcule la moyenne d'une colonne ou groupe de lignes
    MAX() : retourne le résultat maximum
    MIN() : retourne le résultat minimum
    COUNT() : compte le nombre de lignes dans un résultat
    ROUND() : arrondit la valeur

  • ORDER BY

    Permet de trier les lignes (ASC = ascendant ; DESC = descendant)

  • GROUP BY

    Permet de grouper plusieurs résultats et utiliser une fonction sur un groupe de résultat

  • LIMIT

    Permet de spécifier le nombre maximum de résultats que l'on souhaite obtenir

  • HAVING / WHERE

    HAVING permet de filtrer après le GROUP BY alors que WHERE filtre avant
